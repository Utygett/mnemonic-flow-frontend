Что хранится в App
Файл: src/App.tsx
App держит только “верхний уровень”: провайдеры (AuthProvider), роутинг/ветвление по страницам, глобальные оверлеи (PWAUpdatePrompt/OfflineStatus), и переключение главных экранов/табов.
​
App не должен содержать бизнес-логику фич (например, как устроен Dashboard или Study), кроме передачи “коллбеков верхнего уровня” типа refreshStats()/refreshDecks() или навигации.
​
Как раскладывать фичи (паттерн)
Каждая крупная часть приложения = отдельная папка фичи (у тебя это уже src/screens/dashboard/), внутри — разделение на слои “логика vs UI”, чтобы не мешать состояния/загрузки с разметкой.
​
Минимальный набор для каждой фичи:

FeatureContainer.tsx — состояние, запросы, склейка actions/model, выбор экранов.
​
FeatureView.tsx — чистый UI: получает model и actions, не ходит в API и не решает “что грузить”.
​
feature.types.ts — типы Model/Actions и локальные типы фичи.
​
components/ — маленькие UI-части, которые используются только внутри этой фичи.
​
hooks/ или useXxx.ts рядом — хуки, которые нужны только этой фиче (коллокация).
